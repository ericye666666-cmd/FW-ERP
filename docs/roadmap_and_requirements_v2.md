# 开发路线与需求 v2

## 目标

这一版不是简单补功能，而是把系统从“基础进销存原型”升级为真正贴合门店实操的版本。

重点变化：

1. 标签打印后移到调拨前，不再在商品建档阶段强制打印。
2. 门店签收改为“分批上架 + 分批确认”的过程，而不是一次性整单点完。
3. POS 从“减库存”升级成真正的收银体系。
4. cycle 结束时支持整店退仓和部分退仓。
5. 定价从门店临时操作升级为仓库批次级价格控制。
6. 系统需要支持 Safaricom 对账和离线同步。

## 核心开发路线

### 阶段 1：把主业务流做成可用系统

范围：

- 商品建档
- 单独生成并绑定 barcode
- 仓库收货入库
- 调拨单创建、审核、打印
- 调拨前标签打印
- 门店分批签收、分批上架
- 自动差异确认单
- POS 基础收银
- 库存流水与审计日志

目标：

- 仓库和门店真的可以跑日常业务
- 每一步都有单据、有状态、有留痕

### 阶段 2：把业务控制做严

范围：

- 仓库批次级限价策略
- 收银员改价权限与红色预警日志
- POS 按支付方式收银
- T-Report / Z-Report
- 收银员交接日志
- 停电 / 断网下的离线销售

目标：

- 不再依赖口头管理价格
- 现场异常行为可以追责
- 停电和断网不影响营业

### 阶段 3：把 cycle 管理做出来

范围：

- 整店退仓
- 部分退仓
- 先勾选整店再取消明细
- 退回仓库 `RET` 区
- cycle 结束清空门店货架

目标：

- 让两周一个 cycle 的清货流程真正被系统理解

### 阶段 4：把经营和财务联动做出来

范围：

- 调拨建议和分析报告
- 仓库自动生成调拨单建议
- 批次级利润率
- 门店真实线上营业额
- Safaricom / M-Pesa 对账
- 客户 ID 维度的熟客分析

目标：

- 不只看销售额，还能做利润、复购和发货决策

## 新需求拆解

### 1. 标签打印与调拨分析

- 标签打印不再属于商品建档步骤。
- 标签打印应该发生在仓库准备发货到门店之前。
- 系统应支持：
  - 单件标签打印
  - 跟随调拨单一起批量打印
  - 自动生成调拨建议单
  - 自动附带分析结果，供你勾选确认

### 2. 门店签收改为分批上架流程

- 门店一次来货可能是 `5000` 件，不可能逐件确认。
- 实际流程应为：
  - 店员扫码查看同类商品上次摆放位置
  - 店员决定本次上架位置
  - 实际挂货一批后，在手机端确认这一批的数量和架位
  - 系统持续累计本次调拨单的上架确认进度
  - 最终自动生成差异确认单

- 这意味着系统需要支持：
  - `receiving session`
  - `receiving batch`
  - `store rack assignment`
  - `final discrepancy sheet`

### 3. POS 是完整收银体系，不只是减库存

- 必须支持：
  - 现金
  - M-Pesa
  - 多支付方式混合
  - T-Report
  - Z-Report
  - 收银员交接日志
  - 停电 / 断网场景下的离线销售缓存

- POS 必须保留：
  - 收银员
  - 班次
  - 支付方式
  - 实收金额
  - 改价记录
  - 上传同步状态

### 4. cycle 退仓必须支持整店与部分退仓

- 必须支持两种模式：
  - 整店退仓
  - 部分退仓

- 操作上需要：
  - 先勾选整店
  - 再按条件检索取消部分勾选
  - 或只勾选部分货
  - 退仓单打印
  - 退仓到仓库 `RET` 区

### 5. 定价控制上移到仓库批次层

- 员工原则上不允许私自改价。
- 收银员仍保留改价权限用于突发情况，但：
  - 必须记录改价前后价格
  - 必须生成红色预警日志
  - 必须可被你和督导追查

- 定价控制应支持：
  - 在商品发往门店前由仓库端设定价格策略
  - 按大类或批次设“当日最高限价”
  - 如：某服装大类今日最高限价 `400`
  - 第三天可改为 `300`

- 系统还应支持：
  - POS 直接关联批次级库存
  - 逐件或逐批跟踪利润率
  - 不只是统计总销售额

### 6. Safaricom 与离线多店同步

- 需要接 Safaricom 接口，拿到：
  - 真实线上营业额
  - 支付时间
  - 客户 ID

- 这能支持：
  - 线上线下对账
  - 熟客分析
  - 支付真实性核验

- 系统还必须支持：
  - 离线销售
  - 网络恢复后自动同步
  - 多店多仓实时并发
  - 不依赖手工回填

## 当前最值得优先做的 5 件事

1. 把“门店签收”改成“分批上架确认”模型。
2. 把 POS 从简化销售单改成真正的收银台模型。
3. 加入仓库批次级定价与改价预警。
4. 加入整店退仓 / 部分退仓流程。
5. 为 Safaricom 对接和离线同步预留数据结构。
