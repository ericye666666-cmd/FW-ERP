# 05_NEXT_ACTION_PLAN.md

This file defines the recommended next actions for FW-ERP. It is written for owner-level control: GPT Thinking plans and reviews; Codex implements only small GitHub Issues.

---

## 1. Operating rule from now on

Do not continue with scattered local Codex threads.

The standard workflow is:

```text
GPT Thinking reviews project brain
-> GPT writes GitHub Issue
-> Codex implements only that Issue
-> Pull Request is reviewed
-> User validates clickable page or local /app result
-> Merge only after review
```

Codex should not decide product boundaries by itself.

---

## 2. Immediate priority

The next priority is not to add more features. The next priority is to stabilize the project brain and define source-of-truth boundaries.

Immediate order:

1. Confirm current system map.
2. Decide frontend roles: `frontend_prototype` vs `frontend_react_admin`.
3. Write barcode governance specification.
4. Resolve store replenishment / cycle model.
5. Start one backend fact-source cleanup task.

## 4. Recommended Issue 2: create barcode governance spec

### Title

```text
Create barcode governance spec for RAW_BALE, DISPATCH_BALE, STORE_ITEM, BALE_SALES, and LOOSE_PICK
```

### Goal

Stop barcode, token, identity, and template-scope drift across warehouse, store, POS, printing, returns, and bales sales.

### Scope

Create a documentation-only specification under:

```text
docs/project-brain/barcode-governance.md
```

The spec must define:

- barcode type
- generated by which flow
- printed by which template family
- scanned by which role/page/device
- rejected by which role/page/device
- maps to which business object
- whether it can enter POS
- whether it binds `identity_id`
- legacy compatibility rule

Non-goals:

- Do not change code.
- Do not change templates.
- Do not change POS.
- Do not change backend routes.

Acceptance criteria:

- Table covers `RAW_BALE`, `DISPATCH_BALE`, `STORE_ITEM`, `BALE_SALES`, `LOOSE_PICK`.
- POS rejection rules are explicit.
- `template_scope` is explicitly marked as print-only, not business identity.
