# 第一期系统范围

## 目标

第一期先把你当前最核心的业务闭环跑通：

1. 仓库收货入库
2. barcode 标签打印
3. 仓库到门店调拨
4. 门店签收与差异确认
5. 门店库存
6. POS 销售减库存
7. 基础经营看板

## 第一阶段功能模块

### 1. 商品建档

- 建立商品主数据
- 记录 `barcode`、商品名称、供应商、大类、小类、成本价、原始定价、货架位
- 支持一个商品一条唯一 barcode

### 2. 仓库入库

- 录入到货批次
- 录入每个 barcode 的入库数量
- 写入仓库库存
- 记录仓库货架位

### 3. 标签打印

- 根据商品主数据生成标签打印任务
- 支持热敏纸打印
- 支持重复打印，并记录打印次数和操作人
- 当前打印机型号：`得力 DL-720C`
- 当前 barcode 类型：`Code128`
- 标签规格：
  - 服装：`60mm x 40mm`
  - 百货：`40mm x 30mm`
- 标签内容：
  - `barcode`
  - `品名`
- 标签不展示门店架位，避免影响顾客体验；架位信息仅供工作人员扫码查询
- 第一版标签默认采用：
  - 上方显示品名
  - 下方显示 Code128 条码与条码文本
  - 品名优先控制在 `1-2` 行内，过长时自动截断

### 4. 调拨单

- 从仓库向门店创建调拨单
- 调拨单包含商品、数量、来源仓库、目标门店
- 调拨单支持打印
- 调拨单必须先审核，审核通过后才能进入拣货流程
- 支持状态流转：
  - `draft`
  - `pending_approval`
  - `approved`
  - `picking`
  - `dispatched`
  - `received`
  - `discrepancy_confirmed`
  - `cancelled`

### 5. 门店签收

- 门店店长按调拨单确认到货
- 支持少货、多货、错货、损坏差异登记
- 差异确认完成后，门店库存正式入库
- 门店店长可以确认差异
- 差异库存调整必须由区域主管审批后才正式生效

### 6. 销售减库存

- POS 销售成功后，根据 barcode 扣减门店库存
- 销售记录保留 `门店`、`收银员`、`成交价`、`数量`、`时间`
- 服装门店售价由一线人员直接在商品上手写，系统侧保留原始售价和实际成交价

### 7. 基础分析

- 门店营业额
- SKU 销量
- 品类销量
- 门店库存
- 异常库存

### 8. 货架位与上架管理

- 仓库端使用标准货架位编码，例如：
  - `WH-A-03-02-05`
  - `WH-B-02-03-04`
  - `WH-RET-01-02-02`
  - `WH-PK-01-01-01`
- 仓库区域支持：
  - `IN` 入库区
  - `SORT` 分拣区
  - `A` A货成品区
  - `B` B货区
  - `RET` 退货区
  - `PK` 打包区
  - `GEN` 百货区
- 门店货架位由店员通过手机上传和编辑
- 新建门店时，系统必须支持初始化和维护整套门店架位
- 第二个 cycle 退货时，需要支持门店货架清空并回退至仓库退货区

## 网站 / 本地软件 / APP 分工

### 网站端

- 商品建档
- 仓库入库
- 调拨管理
- 调拨审核
- 门店签收管理
- 库存总览
- 销售与经营看板
- 角色与权限管理
- 门店主数据管理

### 本地软件

- 标签打印
- 调拨单打印
- 仓库扫码拣货
- 收银端扫码
- Windows 收银机优先作为第一版 POS 运行终端

### APP

- 店长签收
- 差异确认
- 快速盘点
- 查库存
- 上传或编辑门店货架位

## 第一期开发表顺序

1. 商品建档 + 仓库入库
2. 标签打印
3. 调拨单与状态流转
4. 门店签收与差异确认
5. 门店库存
6. 销售减库存接口
7. 基础报表

## 暂时不放在第一期的内容

- 高级补货预测
- 自动调拨推荐
- 复杂财务报表
- 完整移动端管理功能
- 多仓复杂波次拣货

## 本轮已经确认的业务规则

- 当前仓库数量：`1`
- 当前在营门店：`3`
- 当前门店代码：
  - `UTAWALA`
  - `PAIPLINE`
  - `KINNO`
- 门店将持续新增，门店信息和店长信息必须可配置
- 权限必须支持自定义角色，例如：
  - 仓库员
  - 仓库主管
  - 店长
  - 收银员
  - 区域主管
  - 管理员
- 收银端当前硬件为 Windows 收银机 + 3D 扫码枪
- 调拨审核角色：`仓库主管`
- 第一版 POS 方案：直接在当前 Windows 收银机上运行我们自己的网页 POS
