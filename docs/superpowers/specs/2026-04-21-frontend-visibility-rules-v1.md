# 前端菜单与按钮显示规则 V1

日期：2026-04-21

关联文档：

- [角色权限矩阵 V1 设计](/Users/ericye/Desktop/AI自动化/retail_ops_system/docs/superpowers/specs/2026-04-21-role-permission-matrix-v1-design.md)
- [后端接口鉴权规则 V1](/Users/ericye/Desktop/AI自动化/retail_ops_system/docs/superpowers/specs/2026-04-21-backend-auth-rules-v1.md)

适用范围：当前一期前端只覆盖以下 5 个角色

- `warehouse_supervisor` = 仓库主管
- `area_supervisor` = 区域运营经理
- `store_manager` = 店长
- `store_clerk` = 店员
- `cashier` = 收银员

## 1. 设计目标

前端只负责“显示权限”和“入口管理”，不负责最终授权。

前端必须保证：

- 登录后只显示当前角色可进入的工作区
- 同一设备上不混出不该看到的导航
- 店长端和店员端分离
- POS 只显示收银工作区
- 所有按钮都按角色、设备、数据范围显示

## 2. 当前原型需要先纠正的问题

当前 `frontend_prototype/index.html` 里，顶部工作区是统一展示的：

- `overview`
- `warehouse`
- `operations`
- `store`
- `admin`

这不符合 V1 权限原则。V1 必须改成：

- 先按角色和设备决定“显示哪个工作区”
- 不再把全部工作区对所有人开放

当前已有两个可复用结构：

- `.store-role-panel-manager`
- `.store-role-panel-clerk`

这两个结构是对的，但必须从“样式区分”升级成“真实角色显示区分”。

## 3. 前端总规则

### 3.1 角色切换规则

- 一个账户可以拥有多个角色
- 登录后先拿 `auth/me`
- 如果账户有多个角色，前端只展示当前设备允许进入的角色
- 用户必须显式选择一个角色进入工作区
- 进入角色后，本次会话只显示该角色菜单

### 3.2 设备切换规则

- `desktop`：允许 `warehouse_supervisor` / `area_supervisor` / `store_manager` / `store_clerk`
- `pda`：只允许 `store_clerk`
- `pos`：只允许 `cashier`

### 3.3 隐藏优先于禁用

前端优先做“隐藏”，不是“显示后禁用”。

只有以下情况可用禁用态：

- 页面允许看，但当前状态不能操作
- 例如：收银员未开班，显示“销售页”，但“提交销售”按钮禁用并提示“请先开班”

若角色根本不该看到：

- 直接隐藏入口
- 不显示无权限菜单
- 不显示空白页面

## 4. 角色默认入口

| 角色 | 设备 | 默认首页 | 说明 |
| --- | --- | --- | --- |
| `warehouse_supervisor` | `desktop` | 仓库控制台 | 仓库主作业入口 |
| `area_supervisor` | `desktop` | 区域运营控制台 | 区域调拨、现金巡店、新店筹划入口 |
| `store_manager` | `desktop` | `5. 门店执行控制中心` | 门店签收、分配、异常处理入口 |
| `store_clerk` | `pda` | `7. 店员 PDA 上架工作台` | 当前 bale、定价、打印、上架 |
| `store_clerk` | `desktop` | `7.1 打印/补打支持页` | 桌面支持操作 |
| `cashier` | `pos` | `9. 收银销售页` | POS 唯一主入口 |

## 5. 顶部工作区可见规则

当前原型里的工作区标签需要按下表重构：

| 顶部工作区 | `warehouse_supervisor` | `area_supervisor` | `store_manager` | `store_clerk` | `cashier` |
| --- | --- | --- | --- | --- | --- |
| `overview` | 可见，但只显示仓库相关摘要 | 可见，但显示区域摘要 | 可见，但显示本店摘要 | 不显示 | 不显示 |
| `warehouse` | 可见 | 不显示 | 不显示 | 不显示 | 不显示 |
| `operations` | 不显示 | 可见 | 不显示 | 不显示 | 不显示 |
| `store` | 不显示 | 不显示 | 可见 | 可见 | 不显示 |
| `admin` | 不显示 | 不显示 | 不显示 | 不显示 | 不显示 |
| `pos` | 不在当前原型顶部标签中，V1 应独立出来 | 不显示 | 不显示 | 不显示 | 可见且唯一 |

硬规则：

- `cashier` 不得看到 `overview / warehouse / operations / store / admin`
- `store_clerk` 不得看到 `warehouse / operations / admin`
- `store_manager` 不得看到 `warehouse / operations / admin`
- `area_supervisor` 不得看到 `warehouse / store / admin`
- `warehouse_supervisor` 不得看到 `operations / store / admin`

## 6. 页面级显示规则

## 6.1 仓库主管工作区

仅显示以下页面：

- 入仓 / 总量复核
- bale 条码打印
- 分拣任务创建与管理
- 分拣确认入库
- 仓库库存 / 分拣库存
- 发店打包 / 发运跟踪
- 与仓库相关的打印任务

必须隐藏：

- 区域经营驾驶舱
- 区域现金页面
- 门店签收与分配页面
- 店员 PDA 上架工作台
- POS 收银页面

## 6.2 区域运营经理工作区

仅显示以下页面：

- 区域经营驾驶舱
- 区域调拨 / 补货规划 / 调拨单管理
- 风控预警中心
- 现金收取与存储
- 巡店记录与评分
- 新店筹划 / 开店准备事项
- 退款 / 作废 / 差异 / 现金异常审批视图

可以复用当前原型 `operations` 里的页面，但要删掉不在本轮范围内的内容：

- `门店限价规则`：先隐藏，不在本轮放开
- `AI 固定分析框架`：可保留展示，但不要变成老板大盘
- `会议纪要与 AI 分析`：先隐藏，不在本轮核心权限里
- `区域经理评分卡`：先隐藏，避免混入老板管理视角

必须隐藏：

- 仓库执行页面
- 店员工作台
- POS 销售页

## 6.3 店长工作区

仅显示以下页面：

- `5. 门店执行控制中心`
- 待签收 dispatch bale 列表
- dispatch bale 签收页
- bale 分配给店员页
- 门店进度监控
- 作废单审核
- 顾客退货 / 退款审核
- 支付异常处理
- 本店库存 / 本店销售摘要
- 本店现金交接与收现状态

必须隐藏：

- 店员 PDA 工作台核心操作按钮
- 区域运营管理页面
- 仓库控制台
- POS 销售页

说明：

- 店长可以看店员进度
- 店长不能直接进入“像店员一样批量定价、批量打印、批量上架”的主工作流
- 如需下场干活，应切换到 `store_clerk`

## 6.4 店员工作区

仅显示以下页面：

- `6.2 我的当前包`
- `7. 店员 PDA 上架工作台`
- `7.2 直挂店员工作台`
- `7.1 打印/补打支持页`
- 我的上架会话
- 我的打印任务

必须隐藏：

- 门店签收按钮
- bale 分配按钮
- 整店异常审批页
- 退款/作废审核页
- 支付异常处理页
- 跨店和整店经营摘要
- 区域管理页
- 仓库页
- POS 页

## 6.5 收银员工作区

必须是单独 POS 界面，只显示：

- 开班
- 销售录单
- 作废申请
- 退款申请
- 支付异常登记
- 交班申请
- 关班
- 本班 T-Report / Z-Report
- 本班交款记录

必须隐藏：

- 门店签收
- 店员上架
- 打印/补打支持
- 区域运营页面
- 仓库页面
- 系统管理

## 7. 按钮级显示规则

| --- | --- | --- | --- | --- |

| `发起作废` | 显示 | 隐藏 | 隐藏 | 仅收银员发起 |
| `发起退款` | 显示 | 隐藏 | 隐藏 | 仅收银员发起 |
| `登记支付异常` | 显示 | 隐藏 | 隐藏 | 仅收银员发起 |
| `交班申请` | 显示 | 隐藏 | 隐藏 | 仅当班收银员 |

- 非当班收银员登录 POS，只能显示“本机已被当前班次占用”提示页

## 8. 当前原型的直接改造规则

## 8.1 顶部标签

以下选择器不能再默认全显：

- `.workspace-tab[data-workspace-target="overview"]`
- `.workspace-tab[data-workspace-target="warehouse"]`
- `.workspace-tab[data-workspace-target="operations"]`
- `.workspace-tab[data-workspace-target="store"]`
- `.workspace-tab[data-workspace-target="admin"]`

必须改成：

- 根据 `active_role_code + device_type` 动态渲染
- 不要先渲染再隐藏，避免闪现

## 8.2 门店页 manager / clerk 面板

当前原型已经有：

- `.store-role-panel-manager`
- `.store-role-panel-clerk`

V1 规则：

- `store_manager` 只渲染 `.store-role-panel-manager`
- `store_clerk` 只渲染 `.store-role-panel-clerk`
- 两者不能同屏并列

## 8.3 operations 工作区

当前 `operations` 工作区要收口成 `area_supervisor` 专区。

保留：

- 区域经营驾驶舱
- 业务动作台
- 风控预警中心
- 现金收取与存储
- 巡店记录与评分

隐藏：

- 与老板经营分析直接相关的内容
- 与中国侧业务相关的内容
- 与系统管理相关的内容

新增入口：

- 新店筹划 / 开店准备事项

## 9. 多角色账户的前端行为

### 9.1 店长 + 店员

- `desktop` 登录后可选 `店长` 或 `店员`
- 默认进入 `店长`
- 若切到 `店员`，页面直接切到店员工作区
- 不允许在店长工作区里出现店员主操作按钮

### 9.2 店员 + 收银员

- `pda` 只显示 `店员`
- `pos` 只显示 `收银员`
- 不允许在 POS 上出现店员工作台入口

### 9.3 店长 + 收银员

- `desktop` 只显示 `店长`
- `pos` 只显示 `收银员`
- 不允许因账户兼岗就在桌面端显示 POS 操作按钮

## 10. 前端与后端的职责边界

前端负责：

- 菜单显示
- 页面入口控制
- 按钮显示/禁用
- 多角色切换
- 设备入口分流

后端负责：

- 真正的权限放行
- 数据范围过滤
- 班次校验
- POS 设备校验
- 审批角色校验

前端不得假设：

- “按钮没显示，后端就不用挡”
- “用户不会自己调接口”

## 11. 本文档的落地结论

V1 前端权限不是做一个“全量系统导航”，而是做五个严格分离的角色工作区：

- 仓库主管工作区
- 区域运营经理工作区
- 店长工作区
- 店员工作区
- 收银员 POS 工作区

其中最重要的三条是：

- 店长端和店员端必须彻底分开
- 收银端必须从普通门店界面中隔离出来
- 当前原型的 `warehouse / operations / store / admin` 全显结构必须被角色化渲染替代
