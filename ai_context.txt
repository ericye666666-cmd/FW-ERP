==================================================
1. BUSINESS MODEL
==================================================

Core business:
- Multi-store retail.
- Main business is second-hand apparel.
- Main operating flow is warehouse sorting flow.
- A smaller exception flow exists for already-sorted goods that can go directly to stores.

Additional business lines:
- Department store / general merchandise ("百货") will be added as a separate business line.
- Bulk bale selling ("整包销售流") is a dedicated sale scene, separate from normal store piece-level merchandising.
- Bale selling may consume either raw inbound bales that never enter sorting or bales re-packed from sorted results.
- Some A/B grade goods may be mixed into resale bales and sold by weight.

Store strategy:
- Stores may sell both second-hand apparel and department store goods.
- Current core operational focus is apparel.

4. MAIN FLOW (PRIMARY FLOW)
==================================================

Main flow = sorting flow.
This is the default business flow.
More than 80% of goods are expected to go through this flow.

Step-by-step:

1. China source bale entry
- China side records container type, shipment, supplier, category, subcategory, package count, per-package weight, package price.

2. China cost supplement
- Add head freight, customs, tail freight, currency, amount, payer, payment method, proof files.
- System forms container-level / source-bale-level cost pool.

3. Kenya shipment master creation
- Kenya side selects a China-created shipment.
- Do not freely type shipment if source already exists.

4. Package inbound
- Kenya side records actual inbound by selecting China expected lines.
- Kenya side should not freely type supplier/category here.
- Kenya records actual package count and actual per-package weight.

5. Total confirmation / reconciliation
- Compare China expected vs Kenya actual.
- Confirm differences before moving on.

6. Inbound bale one-code-per-bale
- Generate warehouse-side bale barcode.
- This is for warehouse tracking, not the final store barcode.

7. Create sorting task
- Select bales.
- Bind sorter(s).

8. Sorting task management
- Supervisors view parallel tasks.
- Tasks have start time / end time / running duration.

9. Sorting confirmation inbound
- Record sorting result lines.
- This is where product-level token is generated.
- This is also where single-piece cost is locked for main flow.

10. Store transfer order
- Operations / supervisor decides which sorted goods go to which store.

11. Warehouse picking / dispatch
- Warehouse picks by transfer order.

12. Create store dispatch bales
- Repack goods for a store into new bales.
- These are NOT the original inbound bales.

13. Delivery to store

14. Store signs receipt of dispatch bales
- Store receives store-dispatch bale barcode.

15. Store manager assigns one bale to one clerk
- One dispatch bale can only be handled by one clerk.

16. Store clerk edits sale information
- Clerk fills:
  - sale price
  - rack location
- Clerk may identify high-value pieces and assign higher prices.

17. Store clerk prints final item barcode
- This activates the final physical item identity in store execution.

18. Shelf / in-store placement

19. Sale / price update / rack update / return-to-warehouse

Important:
- Return-to-warehouse is terminal.
- No return-to-warehouse -> re-transfer flow.

==================================================
5. EXCEPTION FLOW (DIRECT-HANG FLOW)
==================================================

Exception flow is only for a small portion of goods that are already sorted and can go straight to store hanging.

Flow:
- China source + cost pool
- Kenya inbound
- 4.5 direct-hang unpacking / counting / cost confirmation
- store transfer
- warehouse dispatch bale creation
- store receives bale
- assign one clerk
- 7.2 direct-hang clerk workbench
- print final item barcode
- shelf

Important direct-hang cost rule:
- China/source bale gives total bale cost.
- Kenya warehouse unpacks and counts total pieces.
- unit_cost = source_bale_total_cost / actual_piece_count
- then transfer to stores by quantity.

==================================================
6. BULK BALE SELLING FLOW
==================================================

This is separate from normal store item flow.

Source paths:
- Raw inbound bales may be sold directly by bale / by weight without entering sorting.
- Sorted results may be recombined into bales and sold by bale / by weight.

Cases:
- Some resale bales mix controlled proportions of A and B grade goods.

Important:
- This is not the same as store piece-level merchandising.
- Treat as a small inventory + outbound + settlement flow.
- Raw bale total stock stays under warehouse main flow until a bale is moved into bale-sales pool.
- Bale sales thread only handles bales already assigned for direct sale or re-packed sale.

Current page:
- 0.4 resale bale work order

==================================================
7. COST MODEL
==================================================

Three levels of cost:

1. China goods value
- package price
- original goods cost

2. Source bale / container cost pool
- goods value
- head freight
- customs
- tail freight
- unified into KES as management currency

3. Piece-level unit cost
- locked when goods become item-level token

Main flow cost lock point:
- 0.2 sorting confirmation inbound

Direct-hang flow cost lock point:
- 4.5 direct-hang unpacking / counting / cost confirmation

Current cost models:
- sorting_pool_avg_v1
  - source pool -> source bale -> total pieces average
- sorting_piece_weight_v2
  - use standard piece weight per category/subcategory for weighted allocation

Important:
- Store staff do not enter cost.
- Store only edits sale price and rack position.

- Switching supplier should switch visible categories.

Language rules:
- China-side pages: supplier / category_main / category_sub show Chinese + English when available.
- Kenya-side pages: only English.

Current key pages using this logic:

10. PRINTING LOGIC
==================================================

Current physical testing target:
- Warehouse bale barcode label printing
- Printer: Deli DL 720C
- Label size currently tested: 60x40 mm
- Use TSPL raw printing
- Barcode type: Code128 1D barcode

Important:
- Do NOT rely on generic macOS A4/Letter printer queue for label output.
- macOS generic queue may show installed but still be wrong for label printing.
- Current working path is TSPL raw printing.

Warehouse bale label human-readable display direction:
- Remove BALE-BL- prefix from human-readable large text.
- Show compact identity parts for people.
- Keep full barcode value for machine scanning.

Printing workbench:
- Printing page is now treated as a separate print station.
- User can lock one shipment and then print categories in chosen order.
- Do not force direct transition into sorting.
- Print by category.
- Support partially completed daily printing.

==================================================
11. STORE EXECUTION RULES
==================================================

Manager side:
- receive dispatch bales
- assign one bale to one clerk
- monitor progress
- handle exceptions

Clerk side:
- work only on assigned bale(s)
- fill sale price and rack
- mark high-value items
- print barcode
- shelf items

Practical rule:
- One hanger usually holds one type/category.
- Clerk tends to hang one category first, then print labels in batch for that category.
- Therefore PDA workbench should support grouped handling, not constant per-piece switching.

==================================================
12. IDENTIFYING THE FINAL ITEM
==================================================

There are multiple identifiers in the system:

1. Inbound warehouse bale barcode
- warehouse tracking only

2. Store dispatch bale barcode
- used for store receipt and assignment

3. Product token / item token
- generated before final store print

4. Final item identity ID / final item barcode
- used after store print
- this is the long-term lifecycle identity

The final item identity must connect:
- supplier
- cost
- store
- print time
- shelf time
- sale price
- future sale
- future price change
- future rack change
- future return-to-warehouse
